The **renderer process** is the React-based user interface layer. Each window runs in its own renderer process, communicating with the main process via IPC.

### Usage Examples

<CodeGroup>
```tsx Tab Management
import { useState, useEffect } from 'react';

function Tabbar() {
  const [tabsData, setTabsData] = useState(null);
  
  // Subscribe to tab updates
  useEffect(() => {
    // Get initial data
    flow.tabs.getData().then(setTabsData);
    
    // Subscribe to changes
    const unsubscribe = flow.tabs.onDataUpdated((data) => {
      setTabsData(data);
    });
    
    return unsubscribe;
  }, []);
  
  // Switch to tab
  const handleTabClick = (tabId: number) => {
    flow.tabs.switchToTab(tabId);
  };
  
  // Close tab
  const handleTabClose = (tabId: number) => {
    flow.tabs.closeTab(tabId);
  };
  
  return (
    <div>
      {tabsData?.tabs.map(tab => (
        <Tab 
          key={tab.id}
          data={tab}
          onClick={() => handleTabClick(tab.id)}
          onClose={() => handleTabClose(tab.id)}
        />
      ))}
    </div>
  );
}
```

```tsx Navigation
function AddressBar() {
  const [url, setUrl] = useState('');
  const [activeTab, setActiveTab] = useState(null);
  
  useEffect(() => {
    flow.tabs.getData().then(data => {
      const active = data.tabs.find(t => t.isActive);
      setActiveTab(active);
      setUrl(active?.url || '');
    });
  }, []);
  
  const handleNavigate = () => {
    if (activeTab) {
      flow.navigation.goTo(url, activeTab.id);
    }
  };
  
  const handleReload = () => {
    if (activeTab) {
      flow.navigation.reloadTab(activeTab.id);
    }
  };
  
  return (
    <div>
      <button onClick={handleReload}>Reload</button>
      <input 
        value={url} 
        onChange={e => setUrl(e.target.value)}
        onKeyDown={e => e.key === 'Enter' && handleNavigate()}
      />
    </div>
  );
}
```

```tsx Spaces
function SpaceSwitcher() {
  const [spaces, setSpaces] = useState([]);
  const [currentSpace, setCurrentSpace] = useState(null);
  
  useEffect(() => {
    // Load spaces
    flow.spaces.getSpaces().then(setSpaces);
    flow.spaces.getUsingSpace().then(setCurrentSpace);
    
    // Subscribe to changes
    const unsubscribe = flow.spaces.onSpacesChanged(() => {
      flow.spaces.getSpaces().then(setSpaces);
    });
    
    return unsubscribe;
  }, []);
  
  const handleSwitchSpace = (profileId: string, spaceId: string) => {
    flow.spaces.setUsingSpace(profileId, spaceId);
  };
  
  return (
    <div>
      {spaces.map(space => (
        <button 
          key={space.id}
          onClick={() => handleSwitchSpace(space.profileId, space.id)}
          className={space.id === currentSpace?.id ? 'active' : ''}
        >
          {space.name}
        </button>
      ))}
    </div>
  );
}
```

## State Management

Flow Browser uses a **hybrid state approach**:

### IPC Subscriptions

<Accordion title="Tab State">
Tab state is managed in the main process and pushed to renderer via IPC:

```typescript
// Subscribe to tab updates
flow.tabs.onDataUpdated((data: WindowTabsData) => {
  // Update React state
  setTabsData(data);
});

// Subscribe to content-only updates (lighter payload)
flow.tabs.onTabsContentUpdated((tabs: TabData[]) => {
  // Update only tab content properties
  setTabs(tabs);
});
```

**Why**: Tab state lives in main process (WebContentsView lifecycle), renderer reflects it.
</Accordion>

<Accordion title="UI State">
UI-specific state (sidebar open, settings panel) uses standard React state:

```typescript
const [sidebarOpen, setSidebarOpen] = useState(false);
const [selectedSetting, setSelectedSetting] = useState('general');
```

**Why**: Purely UI state with no main process involvement.
</Accordion>

<Accordion title="Settings">
Settings are persisted in main process, accessed via IPC:

```typescript
// Read setting
const value = await flow.settings.getSetting('appearance.theme');

// Write setting
await flow.settings.setSetting('appearance.theme', 'dark');

// Subscribe to changes
flow.settings.onSettingsChanged(() => {
  // Reload settings
});
```

**Why**: Settings need to persist across sessions and be accessible to main process.
</Accordion>

## Styling with TailwindCSS 4

Flow Browser uses **TailwindCSS 4** with the new Oxide engine.

<Accordion title="Tailwind Configuration">
Tailwind is configured via Vite plugin:

```typescript
// Vite config
import tailwindcss from '@tailwindcss/vite';

export default {
  plugins: [
    tailwindcss()
  ]
};
```

Custom theme tokens defined in CSS:

```css
@theme {
  --color-primary: oklch(0.5 0.2 200);
  --color-sidebar: oklch(0.95 0.01 200);
}
```
</Accordion>

### Component Styling Pattern

```tsx
import { cn } from '@/lib/utils';
import { cva } from 'class-variance-authority';

const buttonVariants = cva(
  // Base styles
  'inline-flex items-center justify-center rounded-md font-medium transition-colors',
  {
    variants: {
      variant: {
        default: 'bg-primary text-primary-foreground hover:bg-primary/90',
        outline: 'border border-input hover:bg-accent',
        ghost: 'hover:bg-accent hover:text-accent-foreground',
      },
      size: {
        default: 'h-10 px-4 py-2',
        sm: 'h-9 px-3',
        lg: 'h-11 px-8',
      },
    },
    defaultVariants: {
      variant: 'default',
      size: 'default',
    },
  }
);

function Button({ variant, size, className, ...props }) {
  return (
    <button 
      className={cn(buttonVariants({ variant, size }), className)}
      {...props}
    />
  );
}
```

## Animations with Motion

Flow uses **Motion** (Framer Motion v12) for animations.

**Package**: `package.json:91` (motion 12.34.3)

<Note>
Import from `motion/react`, NOT `framer-motion`:

```tsx
import { motion, AnimatePresence } from 'motion/react';
```
</Note>

### Animation Examples

```tsx
import { motion, AnimatePresence } from 'motion/react';

function TabList({ tabs }) {
  return (
    <AnimatePresence mode="popLayout">
      {tabs.map(tab => (
        <motion.div
          key={tab.id}
          layout
          initial={{ opacity: 0, scale: 0.8 }}
          animate={{ opacity: 1, scale: 1 }}
          exit={{ opacity: 0, scale: 0.8 }}
          transition={{ duration: 0.2 }}
        >
          <Tab data={tab} />
        </motion.div>
      ))}
    </AnimatePresence>
  );
}
```

## React Compiler

Flow Browser uses the **React Compiler** for automatic optimizations.

**Plugin**: `package.json:73` (babel-plugin-react-compiler)

The compiler automatically:
- Memoizes components
- Optimizes re-renders
- Eliminates unnecessary useCallback/useMemo

<Tip>
No need for manual memoization in most cases - the compiler handles it!
</Tip>
